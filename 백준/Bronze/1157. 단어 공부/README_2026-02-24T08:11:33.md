# [Bronze I] 단어 공부 - 1157 

[문제 링크](https://www.acmicpc.net/problem/1157) 

### 분류

구현, 문자열

### 문제 설명

<p>알파벳 대소문자로 된 단어가 주어지면, 이 단어에서 가장 많이 사용된 알파벳이 무엇인지 알아내는 프로그램을 작성하시오. 단, 대문자와 소문자를 구분하지 않는다.</p>

### 입력 

 <p>첫째 줄에 알파벳 대소문자로 이루어진 단어가 주어진다. 주어지는 단어의 길이는 1,000,000을 넘지 않는다.</p>

### 출력 

 <p>첫째 줄에 이 단어에서 가장 많이 사용된 알파벳을 대문자로 출력한다. 단, 가장 많이 사용된 알파벳이 여러 개 존재하는 경우에는 ?를 출력한다.</p>

# 💻 코드 리뷰
해당 문제는 알파벳 단어가 주어졌을때 가장 많이 사용된 알파벳을 대문자로 출력하는 문제이며 가장 많이 사용된 알파벳이 여러개라면 ?표를 출력해야한다.  
  
문제를 읽고 어떤 방법으로 접근할까에대한 고민을 해본 결과 모든 문자를 대문자로 변환하여 카운트하기로 결정하였다.  
또 대문자로 변환하는 과정에서 아스키코드를 활용하여 이를 정수형 숫자로 변환, 알파벳 크기만큼의 배열을 선언하여 카운트한다.  
변환된 알파벳을 인덱스로 A,a는 배열의 0번째 인덱스로 카운트하여 저장하는 식이다.  

비슷한 유형의 문제를 풀이한 경험이 존재하여 대문자 A와 소문자 a간의 아스키 코드 숫자간의 차이가 32라는 사실을 알고있었지만   
이를 모른다고 가정하였을때로 접근하여 문제를 풀이하였다.  
배열을 (int)('z'-'a')+1 크기만큼으로 선언하여주고 입력된 문자열을 toCharArray()를 통해 문자로 나누어 진행하였다.  

이후 각 문자에 대문자 A를 빼주고 정수형으로 변환해주면 c가 대문자일경우 맞는 인덱스 값으로 변환되고 소문자일 경우 이상한 값이 저장될 것이다.  
단순하게 toUpperCase()를 통해 문자열 자체를 대문자로 변환하고 문제 풀이를 진행하면 더 간단하지만 해당 메서드를 모른다고 가정하고 아래와 같이 진행하였다.  
c가 소문자일 경우 소문자 a에서 대문자 A를 뺀 정수형 만큼을 기존 index에서 빼주어야한다.   
그이유는 위에서 언급하였듯이 대문자 A와 소문자 a간에는 32만큼의 차이가 존재하는데 'a'-'A'가 정수형으로 계산하였을때 32이다.   
다만 해당 방법으로 문제를 풀이하려면 대문자보다 소문자가 숫자가 더 크다는 것은 알고 있거나 모르겠다면 절대값 처리가 필요할것으로 보인다.  

위의 코드들은 문자열을 문자로 나누고 해당 문자들을 대문자로 변환후 정수처리해 배열의 인덱스로 카운트하는 과정이다.  
해당 과정을 마친 후에는 배열에서 가장 큰 값을 구하고 그 큰 값이 반복되는 지를 카운트해 ?를 출력할지 숫자를 출력할지를 결정해주면된다.  



 ## 🏆 메모 

처음 문제를 읽고나서 문자를 문자열로 나눈후 모든 값을 대문자로 취급해 알파벳 갯수만큼의 배열로 카운트해야겠다고 생각했다.   
하지만 ArrayIndexOutOfBounds 에러가 자꾸 발생해 코드를 확인해 보니 문자를 정수형으로 변환하는 과정에서 별도의 함수를 사용하지 않고 풀이하려다 보니 실수가 발생하였고 
위와 같이 코드를 수정하여 문제를 해결하였다.  

정확히는 문자로 표현된 숫자로 변환할때 문자가 ch라고 했을때 ch-'0'와 같이 사용하면 문자로 표현된 숫자를 온전한 정수형으로  변환하여사용이 가능한데
본인은 해당 문제에서 ch가 문자로 표현된 숫자가 아닌에도 불구하고
ch-'0'과 같이 사용해 문자들간에 뺄셈으로 정확한 값이 나와야하는데 '0'을 한번더 빼버리니 값이 이상하게 나온것이다.  

별도의 정수처리 필요 없이 문자들간에 차를 변환했어야했는데
문자로 표현된 숫자를 변환하는 것과 형변환하는 과정을 혼돈해 발생한 문제였다.
