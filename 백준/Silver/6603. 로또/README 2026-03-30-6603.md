### 문제 설명

<p>독일 로또는 {1, 2, ..., 49}에서 수 6개를 고른다.</p>

<p>로또 번호를 선택하는데 사용되는 가장 유명한 전략은 49가지 수 중 k(k>6)개의 수를 골라 집합 S를 만든 다음 그 수만 가지고 번호를 선택하는 것이다.</p>

<p>예를 들어, k=8, S={1,2,3,5,8,13,21,34}인 경우 이 집합 S에서 수를 고를 수 있는 경우의 수는 총 28가지이다. ([1,2,3,5,8,13], [1,2,3,5,8,21], [1,2,3,5,8,34], [1,2,3,5,13,21], ..., [3,5,8,13,21,34])</p>

<p>집합 S와 k가 주어졌을 때, 수를 고르는 모든 방법을 구하는 프로그램을 작성하시오.</p>

### 입력 

 <p>입력은 여러 개의 테스트 케이스로 이루어져 있다. 각 테스트 케이스는 한 줄로 이루어져 있다. 첫 번째 수는 k (6 < k < 13)이고, 다음 k개 수는 집합 S에 포함되는 수이다. S의 원소는 오름차순으로 주어진다.</p>

<p>입력의 마지막 줄에는 0이 하나 주어진다. </p>

### 출력 

 <p>각 테스트 케이스마다 수를 고르는 모든 방법을 출력한다. 이때, 사전 순으로 출력한다.</p>

<p>각 테스트 케이스 사이에는 빈 줄을 하나 출력한다.</p>


## 풀이 방법

해당 문제는 앞서 풀이했었던 N과 M 시리즈 문제로 dfs를 활용하여 숫자의 조합 경우의 수를 출력하는 문제이다.
하지만 일부 중복을 허용하지만 일부는 허용하지 않는? 그런 규칙을 가지고 있다.  
이렇게 말하는 이유는 뒤에서 설명하도록하겠다.

```java
import java.util.*;

public class Main{
    public static int arr[];
    public static boolean visited[];
    public static StringBuilder sb=new StringBuilder();
    public static void main(String args[]){
        Scanner s=new Scanner(System.in);
        
        int n=s.nextInt();
        int m=s.nextInt();
        
        arr=new int[m];
        visited=new boolean[n];

        dfs(n,m,0);
        System.out.print(sb);
    }
    public static void dfs(int n,int m,int depth){
        if(depth==m){
            for(int i=0;i<m;i++){
                sb.append(arr[i]).append(" ");
            }sb.append("\n");
            visited[arr[0]-1]=true;
            return;
        }
        
        for(int i=0;i<n;i++){
            if(!visited[i]){
                arr[depth]=i+1;
                dfs(n,m,depth+1);
            }

        }
    }
}

```
그래서 처음에는 코드를 위와 같이 작성하였다. 가장 앞에 있는 숫자가 반복을 마치면, 예를 들어 1 해당 숫자를 방문처리해주는 것이다.  
하지만 이렇게 문제를 풀이할 경우 아래와 같은 문제가 발생하게 된다.
n=3,m=3이라고 가정했을때 문제에서 원하는 출력은 
```
1 1 1
1 1 2
1 1 3
1 2 2
1 2 3
1 3 3
2 2 2
2 2 3
2 3 3
3 3 3
```
이런식이지만 현재 코드는 아래와 같이 출력된다.

```
1 1 1 
1 1 2 
1 1 3 
1 2 2 
1 2 3 
1 3 2 
1 3 3 
2 2 2 
2 2 3 
2 3 3 
3 3 3 
```
얼핏보면 문제없는 출력처럼 보이지만  1 2 3이 출력되었으니 1 3 2는 출력되면 안된다.  
위에서 일부 중복을 허용하지만 일부는 허용하지 않는? 그런 규칙이라는 표현을 사용한 이유이기도하다. 

처음에는 dfs 방문처리로 어떻게든 해결해보려했지만 2가지 depth마다 다르게 방문처리를 하는것은 어렵다고 생각해 다른 접근 방법을 찾았다.  
바로 아래와 같이 코드를 작성하는 것이다.  

```java
import java.util.*;

public class Main{
    public static int arr[];
    public static StringBuilder sb=new StringBuilder();
    public static void main(String args[]){
        Scanner s=new Scanner(System.in);
        
        int n=s.nextInt();
        int m=s.nextInt();
        
        arr=new int[m];

        dfs(n,m,0);
        System.out.print(sb);
    }
    public static void dfs(int n,int m,int depth){
        if(depth==m){
            for(int i=0;i<m;i++){
                sb.append(arr[i]).append(" ");
            }sb.append("\n");
            return;
        }
        
        for(int i=0;i<n;i++){
            if(depth!=0&&arr[depth-1]<=i+1){
                arr[depth]=i+1;
                dfs(n,m,depth+1);
            }
            else if(depth==0){
                arr[depth]=i+1;
                dfs(n,m,depth+1);
            }
        }

    }
}

```
문제에서 원하는 출력 규칙을 확인해보자면 depth가 커짐에 따라 오름차순으로 정렬되는 것을 확인해 볼 수 있다.   
이를 별도의 방문처리말고 단순 조건문으로 바꿔서 문제를 해결하였다.

해당 문제를 분석하기전까지 본인은 위의 방식처럼 나누어서 진행하는 것이 최선이라고 판단했다.  
depth마다 방문처리를 다르게 하는 것은 불가능하니까 하지만 그것은 본인이 재귀함수가 Stack과 같은 성질인 FIFO 특징을 가진다는 것을 간과했다.  
위처럼 조건문 대신 아래와 같이 start값을 지정해준다면 자연스럽게 가능한것이었다.  


```java
import java.util.*;

public class Main {
    static int N, M;
    static int[] arr;
    static StringBuilder sb = new StringBuilder();

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        N = sc.nextInt();
        M = sc.nextInt();
        arr = new int[M];

        dfs(0, 1);
        System.out.print(sb);
    }

    static void dfs(int depth, int start) {
        if (depth == M) {
            for (int num : arr) {
                sb.append(num).append(" ");
            }
            sb.append("\n");
            return;
        }

        for (int i = start; i <= N; i++) {
            arr[depth] = i;
            dfs(depth + 1, i); 
        }
    }
}


```
코드도 간결해지고 가독성도 좋아졌다. 불가능하다고 생각했던 이유는 재귀함수가 FIFO인것을 고려하지 못하고  
문제를 풀이했기 때문이라고 생각한다.
