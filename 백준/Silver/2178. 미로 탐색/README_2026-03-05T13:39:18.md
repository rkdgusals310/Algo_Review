<p>미로에서 1은 이동할 수 있는 칸을 나타내고, 0은 이동할 수 없는 칸을 나타낸다. 이러한 미로가 주어졌을 때, (1, 1)에서 출발하여 (N, M)의 위치로 이동할 때 지나야 하는 최소의 칸 수를 구하는 프로그램을 작성하시오. 한 칸에서 다른 칸으로 이동할 때, 서로 인접한 칸으로만 이동할 수 있다.</p>

<p>위의 예에서는 15칸을 지나야 (N, M)의 위치로 이동할 수 있다. 칸을 셀 때에는 시작 위치와 도착 위치도 포함한다.</p>

### 입력 

 <p>첫째 줄에 두 정수 N, M(2 ≤ N, M ≤ 100)이 주어진다. 다음 N개의 줄에는 M개의 정수로 미로가 주어진다. 각각의 수들은 <strong>붙어서</strong> 입력으로 주어진다.</p>

### 출력 

 <p>첫째 줄에 지나야 하는 최소의 칸 수를 출력한다. 항상 도착위치로 이동할 수 있는 경우만 입력으로 주어진다.</p>

# 💻 코드 리뷰

해당 문제는 N x M 크기의 배열로 표현된 미로가 존재하고 미로에서 1은 이동할 수 있는 칸, 0은 이동할 수 없는 칸이라고 한다.   
이때 (1,1)에서 출발하여 (N,M)의 위치로 이동할때 지나야하는 최소한의 칸 수를 구하는 프로그램을 작성하는 문제였다.  
문제를 읽고 탐색해야하는데 최소를 구해야한다는 점에서 BFS알고리즘을 활용하여 문제를 해결하기로 결정하였다.  
먼저 main함수에서는 n과 m을 입력받고 해당 크기만큼의 배열을 선언하고 미로를 구성하는  1과 0값을 받아 해당 배열에 저장해주었다.  
이후 bfs함수를 호출하는 구조이다.  
bfs함수에서는 Queue를 통해서 출발점 지점의 좌표를 저장하고 출발지점의 좌표를 방문처리해주었다. 또 해당 지점의 주변 미로들 중 이동 가능하다면 해당 좌표를 Queue에 삽입하고 방문처리해주는 식으로 코드를 작성하였다. 
여기서 BFS알고리즘은 인접 노드들을 저장하지만 해당 문제에서는 미로에서 이동 가능한 인접 미로들로 이동한다는 차이점이 존재했다. 또 미로는 인접리스트처럼 노드간 연결이되어 다음으로 이동 이런식이 아니라서 
상하좌우중 이동가능하고 인접한 미로칸을 모두 삽입하는 식으로 문제를 해결하였다. 이런식으로 현재 위치에서 상하좌우 탐색 이동 가능한, 즉 1이라면 큐에 삽입하고 방문처리이를 반복하다보면 n,m지점에 도달할 것이고   
이때가 최소값이기 때문에 return해주는 식으로 함수가 작동한다.

BFS 알고리즘은 시작 노드를 큐에 삽입하고 방문처리후 큐에서 노드를 꺼내 해당 노드와 인접한 노드중 방문하지 않은 노드를 모두 큐에 삽입하고 방문처리하는 과정을 반복하는 것이기 때문에 위와 같이 작성하였다.  

#  🚀  추가 코드 

```diff
import java.util.*;

public class Main{
    public static int map[][];
    public static void main(String args[]){
        Scanner s=new Scanner(System.in);
        
        int n=s.nextInt();
        int m=s.nextInt();
        map=new int[n][m];

        for(int i=0;i<n;i++){
            String str=s.next();
            for(int j=0;j<m;j++){
                map[i][j]=str.charAt(j)-'0';
            }
            
        }
        bfs(0,0);
        System.out.print(map[n-1][m-1]);
    }
    
    public static void bfs(int x,int y){
        Queue<int[]> queue=new LinkedList<>();
        queue.add(new int[]{x,y});
        int dx[]={-1,1,0,0};
        int dy[]={0,0,-1,1};
        
        while(!queue.isEmpty()){
            int num[]=queue.poll();
            
            
            for(int i=0;i<4;i++){
                int nx=num[0]+dx[i];
                int ny=num[1]+dy[i];
                if(nx>=0&&ny>=0&&nx<map.length&&ny<map[0].length){
                    if(map[nx][ny]==1){
                        map[nx][ny]=map[num[0]][num[1]]+1;
                        queue.add(new int[]{nx,ny});
                    }
                }
            }
            
            
        }
    }
}
```

추가 코드는 visited 함수 없이 탐색하는 방식으로 
```
 if(map[nx][ny]==1){
		map[nx][ny]=map[num[0]][num[1]]+1;
		queue.add(new int[]{nx,ny});
}
```
본 코드를 통해 방문처리와 카운트를 동시에 한다고 볼 수 있다.  
처음 코드에서는 큐에 x,y값을 제외하고 이동할때 카운트하는 cnt를 넘겨주어 칸 이동을 계산해주었다.   
하지만 해당 코드에서는 미로에서 이동하면서 1이 아니면 이동할 수 없기때문에   
이를 활용하여 값을 이동할때마다 이전값을 더해주는 방식으로 방문처리를 표시해주는것이다. 
예를 들어 (1,1)이 1이고 (1,2)가 1이라면 (1,1)에서 (1,2)로 이동 후 (1,2)=2로 표시해 방문한적이 있다라는 것을 표시하는 것이다.
하지만 이런방식은 BFS가 시작점으로 돌아올 확률이 적긴하지만 시작지점은 별도의 방문처리를 해주지 않기 때문에 문제가 될 수도 있겠다는 생각을 했다.
