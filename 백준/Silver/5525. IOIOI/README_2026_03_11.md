### 문제 설명

<p>N+1개의 <code>I</code>와 N개의 <code>O</code>로 이루어져 있으면, <code>I</code>와 <code>O</code>이 교대로 나오는 문자열을 P<sub>N</sub>이라고 한다.</p>

<ul>
	<li>P<sub>1</sub> <code>IOI</code></li>
	<li>P<sub>2</sub> <code>IOIOI</code></li>
	<li>P<sub>3</sub> <code>IOIOIOI</code></li>
	<li>P<sub>N</sub> <code>IOIOI...OI</code> (<code>O</code>가 N개)</li>
</ul>

<p><code>I</code>와 <code>O</code>로만 이루어진 문자열 S와 정수 N이 주어졌을 때, S안에 P<sub>N</sub>이 몇 군데 포함되어 있는지 구하는 프로그램을 작성하시오.</p>

### 입력 

 <p>첫째 줄에 N이 주어진다. 둘째 줄에는 S의 길이 M이 주어지며, 셋째 줄에 S가 주어진다.</p>

### 출력 

 <p>S에 P<sub>N</sub>이 몇 군데 포함되어 있는지 출력한다.</p>

## 풀이 방법

해당 문제 N+1개의 I와 N개의 O로 이루어져 있교, I와 O이 교대로 나오는 문자열을 P(N)라고 할때
I와 S로 이루어진 문자열 S와 N이 주어진다면 S안에 P(N)이 몇군데 포함되어 있는지 를 구하는 프로그램을 작성하는  문제이다.

처음 문제를 읽고 입력 받아야하는 N과 M 의 범위가 작지 않고 시간 제한이 1초이기 때문에 
이중 반복문을 통한 비교 대신 for + substring 을 통해 문제를 해결하려하였다.

```java
import java.util.*;

public class Main{
    public static void main(String args[]){
        Scanner sc=new Scanner(System.in);
        
        int n=sc.nextInt(); // N
        int cnt=0; // S안에 P(N)의 개수
        String str="IOI"; //문자열 P(N)
        for(int i=1;i<n;i++){
            str+="OI";
        }
        
        int m=sc.nextInt(); // M
        String s=sc.next();// 문자열 S

        for(int i=0;i<=m-str.length();i++){
            if(s.substring(i,i+str.length()).equals(str))cnt++;
        }
        
        System.out.print(cnt);
    }
}
```

n을 입력 받고 P(N)의 값을 변수 str에 저장한다. 
P(N)은 처음 시작이 I인것을 제외하면 N의 값이 증가할때마다 "OI"를 반복한다.
이렇게 str에는 P(N)의 값이 저장되고 S를 입력 받아 for +substring를 통해 구간마다 P(N)이 존재 여부를 확인한다.
만약 존재한다면 cnt의 값을 증가시켜주고 이후 cnt를 최종 출력하는 구조이다.

하지만 결과적으로 해당 코드는 50점을 받아 틀린 코드이다. 정확하게 어느 부분이 틀렸다고 나와있지 않지만 
예상해보자면 해당 문제의  조건이 2개가 존재하는데 그중 첫 번째 조건인 (N ≤ 100, M ≤ 10 000) 만을 만족한 것으로 생각된다.
첫번째 조건에서는 위의 코드가 돌아가지만 본 문제의 제한 조건은 1 ≤ N ≤ 1,000,000 , 2N+1 ≤ M ≤ 1,000,000이다.
그래서 시간초과를 예상해 이중 반복문을 사용하지 않고 해결하려고 시도했었지만 substring의 시간복잡도가 O(N)인것을 간과하였다.

for-substring를 사용시 이중 반복문 구조는 아니지만 결과적으로 사용한것이나 마찬가지인 구조인것이다.

그래서 S에서 P(N)의 개수를 구하는 것이 아닌 S에서 P(N)의 규칙을 가진 문자열의 길이를 측정하고 
이후 cnt를 계산하는 방식으로 수정하여 문제를 해결하였다.

코드의 동작 방식은 다음과 같다.
1. n,m,s를 입력받는다.
2.  cnt, start, index,t를 선언한다.
    - cnt : S안에 P(N)의 개수_ 최종적으로 요구하는 값
    - start: S안에 P(N)의 규칙을 가지는 연속적인 문자열의 길이를 저장하는 변수
    - index: 반복문안에서 s를 문자로 나누는 과정에서 현재 s문자열의 몇번째 문자인지를 저장하는 변수
    - t: start를 최종값으로 계산하는 과정에서 계산값을 저장하는 변수

3. while문에서 index가 s의 길이보다 커질때까지 반복한다.
4. ch를 선언하고 s.char(index)를 통해 현재 위치의 문자로 초기화해준다.
5. P(N)의 규칙에 맞게 비교하려면  'I'로 시작해야 한다.
'''java
 if(start==0&&ch=='I'){
            start=1;
            index++;
}
'''
위에서 설명하였듯이 start가 규칙을 가진 문자열의 길이를 저장한다고하였다.
start==0이라함은 곧 길이가 0,첫번째를 의미한다. 그리고 ch가 'I'이면 
규칙이 성립함으로 start=1, index를 증가시켜 다음 문자로 넘어간다.

6.만약 start=0이지만 ch가 'O'라면 index만 증가시켜 다음 문자로 넘어간다. 
 7. 5번과 6번의 경우가 아니라면 아래와 같이 3가지 경우를 나누어 진행한다.
```java
else{
        if(start%2==0&&ch=='I'){index++;start++;}
        else if(start%2==1&&ch=='O'){index++;start++;}
        else{
                    t=(start-1)/2;
                    if(t>=n){
                                cnt+=t-n+1;
                    }
                    
                    start=0;
        }
}
```
참고) P(N)은 처음 'I'를 시작으로 "OI"를 반복하기 때문에 짝수일 때는  'I' 홀수일 때는 'O'가 된다.

 - start의 값이 짝수이면서 ch='I'인 경우
     -  index값을 증가 시키고 start값도 증가

 - start의 값이 홀수이면서 ch='O'인 경우
     - index값을 증가 시키고 start값도 증가

 - 위의 모두 해당하지 않는 경우, 즉 P(N)의 규칙을 만족하지 못한 경우
     - t에 (start-1)/2값을 저장해주고 t가 n보다 크거나 같다면 cnt에 t-n+1값을 저장한다.
 
          ```
          t에서 (start-1)/2를 해주는 이유는 start는 P(N)의 규칙을 반복하는 문자열의 길이이다.  
          P(N)은  N이 증가함에 따라 "IO"를 반복하는 것이 아니라 "OI"를 반복한다. 
          그래서 start-1에서 처음 문자 'I'를 제거하고 2로 나누어 "OI"를 묶어주는 것이다.
          그러면 t는 "OI"가 몇개인지를 나타낸다.
          그리고 n을 빼주면 해당 구간에서 마지막으로 반복할 수 있는 위치가 나오고 
          해당 위치도 포함해야하기 떄문에 1을 더해주는 것이다.
           
    - 계산을 마치면 start을 0으로 바꾸어주어 다시 규칙에 맞는 반복문 길이를 카운트 가능하게 하였다.
    - 해당 위치까지 오면서 어차피 문자 반복 비교인데 for문을 사용하지 않고 
       while 문+ index를 사용하였는지에 대한 의문이 있을 수도 있다.

    - 해당 코드에서 while+ index를 사용하는 이유는 P(N)의 규칙에 맞춰서 start를 카운트 하다가 규칙에 맞지 않으면 계산하고
       다음 문자로 넘어가는 것이 아니라 규칙이 맞지 않은 문자부터 시작해야 한다. 
      
      하지만 for문을 사용하면 값이 자동으로 증가하기 때문에 index를 통해 값을 조정해주었다.
      예를 들어 "IOIOIIOI"인 경우 "IOIOI"와 "IOI"가 포함되어 있는데 index를 사용하지 않았다면 
      "IOIOII"에서 규칙이 맞지 않아 else로 들어와 계산하고 start가 0 그리고  다음 문자로 넘어가 계산한다. 
      그렇다면 "IOIOI"는 맞게 카운트하겠지만 "I"는 넘어가고 "O"에서 비교를 시작하기 때문에 의도한 값이 나오지 않는다. 


 8. 이후 반복문이 종료되고 start에 값이 남아 있을 수 있기 때문에 아래 코드를 통해 최종 계산을 마무리해준다.
```java
   t=(start-1)/2;
   if(t>=n){
               cnt+=t-n+1;
   }
 ```
해당 과정을 마치면 문자열 S에 P(N)이 몇 개 포함되어 있는지 계산할 수 있다.

이런 방식으로 문제를 풀이하면 이중 반복문 없이 문제를 해결 가능하고 시간 복잡도도 O(N)으로 충분히 통과 가능하게 된다.

## 어려웠던 점

문자열에서 문자의 구간을 카운트한다는 생각이 가장 어려웠다. 
입력 범위가 커서 이중 반복문 구조를 피하기 위해
문자열이라 contains나 substring을 통해 쉽게 풀이하려 했지만 
해당 방식도 결론적으로 시간복잡도는 이중 반복문을 사용한 것 과 같았기 때문에 시간이 오래 걸렸다.

해당 아이디어만 생각해낸다면 어렵지 않게 문제를 해결할 수 있을 것으로 생각된다.
