# [Silver V] 그룹 단어 체커 - 1316 

[문제 링크](https://www.acmicpc.net/problem/1316) 

### 분류

구현, 문자열

### 문제 설명

<p>그룹 단어란 단어에 존재하는 모든 문자에 대해서, 각 문자가 연속해서 나타나는 경우만을 말한다. 예를 들면, ccazzzzbb는 c, a, z, b가 모두 연속해서 나타나고, kin도 k, i, n이 연속해서 나타나기 때문에 그룹 단어이지만, aabbbccb는 b가 떨어져서 나타나기 때문에 그룹 단어가 아니다.</p>

<p>단어 N개를 입력으로 받아 그룹 단어의 개수를 출력하는 프로그램을 작성하시오.</p>

### 입력 

 <p>첫째 줄에 단어의 개수 N이 들어온다. N은 100보다 작거나 같은 자연수이다. 둘째 줄부터 N개의 줄에 단어가 들어온다. 단어는 알파벳 소문자로만 되어있고 중복되지 않으며, 길이는 최대 100이다.</p>

### 출력 

 <p>첫째 줄에 그룹 단어의 개수를 출력한다.</p>

# 💻 코드 리뷰

해당 문제는 전형적인 문자열 기반 구현 문제로, 처음 문제를 읽고 문자열을 문자 단위로 순회하면서 알파벳 26개 크기의 배열을 사용해 등장 이력을 기록하고,  
문자가 연속인지 여부를 판단해 그룹 단어를 판별하는 방식으로 접근했다.  

본 문제의 핵심은 각 단어가 그룹 단어인지 아닌지를 확인한 뒤 그룹 단어의 개수를 출력하는 것이다.   
그룹 단어란 어떤 문자가 한 번 등장하면 그 문자가 끊기지 않고 연속해서만 등장해야 하는 단어를 의미하며, 즉 각 문자는 등장 구간이 한 번만 존재해야 한다.  
예를 들어 happy는 p가 연속으로 등장하고 다른 문자들도 한 번씩만 등장하므로 그룹 단어지만, aba는 a가 등장했다가 b로 끊긴 뒤 다시 a가 등장하므로 그룹 단어가 아니다.  

이를 판별하기 위해 문자열을 처음부터 끝까지 비교하면서 이전 문자를 저장하는 previous 변수를 사용했고, 문자의 등장 이력을 기록하기 위해 group[26] 배열을 사용했다.  
먼저 첫 문자는 이전 문자도 없고 방문 기록도 없으므로 group 배열에 방문 처리하고 previous에 저장해 둔 뒤, 다음 문자부터 순회하며 검사했다.  
순회 중 현재 문자와 이전 문자가 같으면 연속 구간이므로 그대로 진행하고, 현재 문자와 이전 문자가 다를 때만 새로운 문자 구간이 시작된 것으로 보고 group 배열을 확인한다.  
이때 group[c-'a']==0이면 처음 등장한 문자이므로 방문 처리한 뒤 previous를 갱신하고 계속 진행하지만,  
group[c-'a']==1이면 과거에 이미 등장했던 문자가 끊겼다가 다시 등장한 것이므로 그룹 단어가 아니라고 판단해 cnt를 증가시키고 반복문을 종료했다.  
마지막으로 cnt==0인 경우에만 answer를 증가시키며, answer는 최종적으로 그룹 단어의 개수를 의미한다.  




 ## 🏆 메모 

문제를 읽고  알파벳의 개수만큼의 크기를 가진 배열을 선언하여 해당 알파벳을 방문여부를 판단하고 이전 문자를 저장하는 변수를 선언해주었다.  
이후 이전 문자와 같은 경우와 같지 않은 경우로 나누어 조건문을 활용하여 문제를 해결해야겠다고 생각 후 풀이를 작성하였다.
